## Summary

Led UI implementation for Facebook social authentication as part of an identity tiger team, integrating OAuth2 flows, account linking, and secure redirects to reduce onboarding friction.

## Problem & Goal

Reduced onboarding friction and improved sign-in/registration success by enabling users to authenticate using their Facebook identities.

## Responsibilities

- Implemented UI flows for social authentication and account linking.
- Integrated with Facebook OAuth APIs and handled edge cases (existing linked accounts, unverified emails, account recovery).
- Coordinated rollout experiments and monitored KPIs.

## Technical details

- OAuth2 authorization redirects, secure token handling, and backend identity integration.
- UI implementation via Marko; Node.js backend integration for account linking and session creation.

## Impact

- Improved registration and sign-in completion rates and reduced friction for new users.
